Partially Addressed (Gaps Within Existing Features)

Brand consistency across campaigns Partial
Brand Kit stores name, logo, colors, watermark — but the watermark/logo is NOT burned into exports yet.
What exists: BrandKit type, BrandKitPanel UI, brandKitToBrief() converts to prompt text
What's missing: video-export.ts doesn't composite the logo/watermark onto frames. The brand kit only influences prompts, not the actual visual output. FFmpeg.wasm can do this with an overlay filter — roughly 30 lines of code.
File: brand-kit.ts, video-export.ts
Undo/redo for scene editing Partial
Undo stack is built but not wired into the main page orchestrator.
What exists: undo.ts has a complete generic UndoStack<T> with 30-entry history
What's missing: page.tsx manages all state via individual useState hooks and never calls pushState/undo/redo. Need to wrap scene mutations with undo tracking.
File: undo.ts, page.tsx
Shopify product feed integration Partial
Single product URL scraping works, but bulk catalog import from Shopify doesn't exist.
What exists: scrape/route.ts handles individual Shopify JSON API + JSON-LD + OG + meta extraction
What's missing: No Shopify Admin API or Storefront API integration for bulk product pulls. Sellers with 200+ SKUs need "import my whole catalog, generate videos for all of them." Shopify's REST admin API at /admin/api/products.json returns paginated product lists — would need OAuth or private app token.
File: scrape/route.ts
Multi-language / localization Partial
TTS voices include some multilingual options, but no real localization pipeline exists.
What exists: Kokoro has 10 voice options, some with accent variants
What's missing: No prompt translation, no locale-specific template variants, no RTL support, no multilingual voiceover with accurate lip-sync. Competitors are weak here too — Creatify's avatar lip-sync fails across languages. Could be a differentiator if done well.
File: audio.ts, prompt-engine.ts
Project management for teams Partial
Project save/load exists, but collaboration, sharing, and approval workflows don't.
What exists: project-io.ts exports .cutagent.json files, Supabase auth + project storage API routes
What's missing: No multi-user projects, no comment/approval system, no client review links, no version history. Agencies managing 10+ client accounts need role-based access and approval gates.
File: project-io.ts, auth.ts, projects/route.ts

Not Addressed (Critical Gaps)

Text overlays, captions, CTAs Missing
No text-on-video capability at all. Every competitor has this. Sellers need "SHOP NOW" buttons, price callouts, and auto-captions.
Why it matters: Meta's own data shows captioned video ads have 12% higher conversion. TikTok's native text overlays are expected in ads. Without text, CutAgent outputs look like raw AI footage, not finished ads.
Implementation path: FFmpeg drawtext filter or HTML5 Canvas compositing. Could also use ASS/SRT subtitle burn-in for auto-captions from voiceover text. Estimate: 3-5 days for basic implementation.
Competitors with this: Creatify Pencil AI Predis AI InVideo AI
AI talking heads / avatars Missing
No avatar or talking head generation. This is the #1 feature competitors lead with for UGC-style ads.
Why it matters: UGC ads with faces convert 2-3× better than pure product footage. Agencies need "virtual creators" to scale without hiring talent. Arcads and Creatify both lead with avatar generation.
Implementation path: TTS pipeline already exists → feed audio to fal.ai avatar endpoint (Kling Avatar v2, MuseTalk, HeyGen). ~100-150 lines of adapter code, 2-3 days. Previous feasibility analysis confirmed this is straightforward.
Competitors with this: Arcads Creatify HeyGen Oxolo
Direct Meta / TikTok upload Missing
No ad platform integration. Users must download MP4, then manually upload to Ads Manager.
Why it matters: Every extra step in the workflow = drop-off. Creatify has direct platform push. The generate→approve→publish loop should be seamless.
Implementation path: Tier 1 (1-2 days): encode to platform specs + deep-link to upload page. Tier 2 (3-6 weeks): Meta Marketing API + TikTok Marketing API OAuth integration. Start with Tier 1.
Competitors with this: Creatify Pencil AI
Performance analytics / creative scoring Missing
No way to predict which creative will perform before launching. No feedback loop from ad performance.
Why it matters: Agencies need to justify spend to clients. Sellers waste budget on underperforming creatives. A "predicted engagement score" or integration with Meta Ads reporting would close the feedback loop.
Implementation path: Phase 1: basic heuristic scoring (hook strength, pacing, CTA presence). Phase 2: Meta Ads API reporting integration to pull ROAS/CPM data back. Phase 2 requires same OAuth approval as direct upload.
Competitors with this: Pencil AI Marpipe
Shoppable video / interactive elements Missing
No tap-to-buy overlays, product links, or interactive hotspots in exported videos.
Why it matters: Shopify and TikTok Shop both support shoppable video. The conversion lift from in-video product tags is significant. However, this is more of a platform feature than a video editor feature — the video file itself can't contain interactive elements, but the metadata/export can be formatted for TikTok Shop or Instagram Shopping.
Implementation path: Export metadata alongside video that maps timestamps to product URLs. Format for TikTok Shop's video upload API. Lower priority — this is a "nice to have" after core gaps are filled.
Auto-captions / subtitle generation Missing
No automatic caption burn-in from voiceover text. Social video without captions loses 80% of viewers who watch muted.
Why it matters: 85% of Facebook video is watched without sound. Auto-captions are table stakes for social video ads. You already have voiceoverText per scene — the transcript is literally sitting there.
Implementation path: Use existing voiceoverText → generate SRT timestamps from TTS duration → burn in via FFmpeg ASS/SRT filter during export. Could also use word-level timestamps from Kokoro/ElevenLabs for karaoke-style captions. Estimate: 2-3 days.

CutAgent's Unique Advantages (No Competitor Has These)

Multi-model orchestration Unique
8 AI video models with smart routing per scene role. No competitor lets you mix Kling for action, Veo for product, Luma for dreamy — in the same video. Every other tool is locked to one model.
Browser-only architecture Unique
Zero backend for generation. FFmpeg.wasm for client-side stitching. No upload queues, no server costs scaling with users. Competitors all run server-side pipelines that limit throughput.
Open-source + transparent pricing Unique
No credit systems, no expiring points, no hidden costs. Users see exact $/sec per model. The confusing billing models of Creatify ($60 for 2 min) and Predis (expiring credits) are the #1 complaint on Trustpilot.

Prioritized Build Roadmap

Ranked by: (user demand × competitive pressure × implementation speed)

1

Text Overlays + Auto-Captions

Without text, your output looks like raw AI footage, not a finished ad. This is the single biggest reason a seller would pick Creatify over CutAgent today. Auto-captions from existing voiceoverText data is low-hanging fruit — the transcript already exists.

⏱ 3-5 days 📊 Highest impact on conversion 🔧 FFmpeg drawtext + ASS subtitles
2

AI Talking Heads

UGC-style ads with faces are the dominant format on TikTok and Reels. Your TTS pipeline is already built — you just need to route audio through a fal.ai avatar endpoint. This turns CutAgent from "product video maker" into "UGC ad maker."

⏱ 2-3 days 📊 Unlocks entire UGC market 🔧 ~150 lines, existing adapter pattern
3

Brand Kit → Actual Export (logo/watermark burn-in)

The Brand Kit UI exists but doesn't actually put logos on videos. This is a promise without delivery — users will feel deceived. Wire the FFmpeg overlay filter to composite logo + watermark during export.

⏱ 1 day 📊 Completes existing feature 🔧 ~30 lines in video-export.ts
4

Smart Export + Platform Deep-Link

Encode to Meta/TikTok specs (H.264, AAC, correct bitrate) and open the upload page directly. Removes the download-then-upload friction without needing API approval.

⏱ 1-2 days 📊 Removes biggest workflow friction 🔧 FFmpeg encoding presets + window.open()
5

Wire Up Undo/Redo

The undo stack is fully implemented but never connected to the UI. Users editing 4-8 scene storyboards will accidentally lose work. Just wire pushState/undo/redo into the existing useState mutations in page.tsx.

⏱ 0.5 day 📊 Prevents user frustration 🔧 Integration only, logic exists
6

Shopify Bulk Catalog Import

Move from single-URL scraping to "connect your Shopify store, generate videos for your whole catalog." Requires Shopify Admin API OAuth. Big unlock for power sellers with 200+ SKUs but not critical for launch.

⏱ 1-2 weeks 📊 Power user feature 🔧 Shopify Admin API + OAuth flow
7

Team Collaboration + Approval Workflows

Agencies managing client accounts need role-based access, review links, and approval gates. This is a SaaS-tier feature — not needed for open-source launch but essential for monetization.

⏱ 2-4 weeks 📊 Agency tier unlock 🔧 Supabase RLS + shared projects
8

Performance Analytics + Creative Scoring

Predict which creative will perform before spending ad budget. Start with heuristic scoring (hook strength, pacing, CTA presence), then integrate Meta Ads reporting for real ROAS feedback.

⏱ 3-6 weeks 📊 Closes the feedback loop 🔧 Meta Ads API + scoring model

Bottom Line

CutAgent addresses 14 of 25 major pain points that e-commerce video ad creators face — and several of those (multi-model orchestration, transparent pricing, browser-only architecture) are genuine differentiators no competitor offers.

The product's core generation pipeline is strong. Where it falls short is in the last mile: the gap between "raw AI video" and "finished ad ready to publish." Text overlays, auto-captions, and talking heads are the three features that would close this gap. All three are implementable in under two weeks using existing infrastructure (FFmpeg.wasm, fal.ai adapters, voiceover text data).

The honest assessment: CutAgent today is a powerful video generation engine that's 80% of the way to being a complete ad creation tool. The remaining 20% is mostly post-production polish (text, captions, branding burn-in) and one new generation type (avatars). These are days of work, not months.
