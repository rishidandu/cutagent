System Layers

The app is a single Next.js frontend with no backend for generation. All AI calls go directly from the browser to fal.ai. The only server-side code is 3 thin API routes for URL scraping and image proxying.

How it works

The main page.tsx (866 lines) is the orchestrator — it holds all state (scenes, style context, audio tracks, API key) and passes it down. There's no state management library; React useState handles everything. State is persisted to localStorage on every change so projects survive page refreshes.

Style Engine — Visual Consistency System
style-engine.ts + frame-extractor.ts + prompt-engine.ts = 714 lines
ENGINE
The core differentiator — chains frames between scenes for cross-model visual consistency

Style Harness Pipeline

Scene N completes
→
Extract last frame (canvas API)
→
Upload to fal.storage
→
Feed as img2vid ref to Scene N+1

Three subsystems

1. Frame Extractor (frame-extractor.ts, 183 lines): Extracts frames from video using HTML5 Canvas. Has a "best frame" scorer that samples 5 positions and picks the one with highest luminance variance (sharpest, most detailed). Also extracts last frame for chaining.

2. Style Engine (style-engine.ts, 105 lines): Orchestrates the chaining. After each scene completes, it extracts the last frame, adds it to the reference bank (replacing stale entries), and plans generation order — sequential if chaining is on, parallel if off.

3. Prompt Engine (prompt-engine.ts, 396 lines): The most sophisticated module. Generates model-optimized prompts per scene role (hook/problem/solution/proof/CTA). Has UGC and Studio style presets, role-specific visual templates, model-aware formatting (Kling=detailed, MiniMax=concise), and anti-hallucination fidelity rules to prevent models from adding fake text to products.

Generation planning

planGenerationOrder() returns batches: with Style Harness on, each scene is its own batch (sequential). With it off, all scenes run in parallel. Hybrid mode is supported but not yet exposed in UI.

Model Adapter Layer — Multi-Model Orchestration
model-adapters.ts (271 lines) + fal.ts (218 lines)
ENGINE
Normalizes 8 different model APIs into one unified interface via adapter pattern

Adapter Pattern

Each model (Kling, Veo, MiniMax, Wan, Seedance, Hunyuan, Luma) has its own adapter function that translates a universal AdapterInput into model-specific fal.ai endpoint + parameters. A regex-based registry routes model IDs to the correct adapter.

Key differences handled: MiniMax doesn't accept aspect_ratio (hint goes in prompt instead). Wan needs num_frames not duration. Veo needs duration as "5s" string. Seedance has generate_audio toggle. MiniMax uses subject_reference for character refs vs image_url for others.

fal.ai Client (fal.ts)

Queue-based generation: submit → poll every 3s → fetch result. Handles image hosting (CDN detection, CORS proxy, data URL upload). Upscales thumbnail URLs from Shopify/Amazon/Demandware CDNs to full-size (fal.ai requires min 300×300). Max 15-minute timeout with 300 polls.

Content Generation — Storyboards, Templates, Audio
storyboard-generator.ts + templates.ts + audio.ts = 724 lines
ENGINE
Auto-generates ad storyboards from product data, applies templates, handles TTS + music

Storyboard Generator

Takes scraped product data → produces 4-scene ad (Hook → Solution → Proof → CTA). Each role maps to a preferred model: Kling for action hooks, MiniMax for proof, Seedance for CTA with audio. Auto-generates voiceover scripts with word budgets based on scene duration (~2.8 words/sec). Picks from 5 different "ad angles" (convenience, quality, social proof, value, unboxing) randomly for variation.

Template System

5 templates: UGC Ad (9:16), Product Showcase (16:9), Explainer (16:9), Before/After (9:16), Social Proof (9:16). Each template assigns optimal model per scene and pre-fills prompts with [PRODUCT] placeholders. Duration clamping ensures template durations match model capabilities.

Audio System

TTS via Kokoro (10 voices, $0.02/1K chars) or ElevenLabs (6 voices, $0.05/1K chars). Music via CassetteAI or Stable Audio 2.5. Auto-fits voiceover to scene duration: if text is slightly over, speeds up to 1.3×; if way over, truncates at word boundary. All audio goes through fal.ai queue system.

Server-Side Routes — Scraping & Proxying
3 API routes in src/app/api/
API
The only server-side code — scrapes product URLs and proxies images past CORS

Routes

  • /api/scrape (505 lines) — Product URL scraper. Tries Shopify JSON API first (/products/[handle].json), falls back to HTML meta + JSON-LD + OpenGraph extraction. Handles Shopify, Amazon, Etsy, eBay, and generic sites. Extracts title, description, price, images, brand, category, color, material.
  • /api/proxy-image — CORS bypass. Many e-commerce CDNs (Shopify, Demandware) block browser cross-origin requests. This route fetches the image server-side and returns the blob.
  • /api/normalize-images — Image preprocessing for fal.ai compatibility.

Entity Extraction (in scrape route)

Goes beyond basic meta tags. Parses JSON-LD @type: "Product" blocks to extract structured entities: category, color, material, keywords. Falls back to keyword matching against known lists (60+ colors, 16 materials, 23 product shapes). These entities feed the prompt engine for much richer, more accurate prompts.

Infrastructure — Export, Persistence, Cost Tracking
video-export.ts + project-io.ts + cost-tracker.ts + job-recovery.ts + undo.ts
INFRA
FFmpeg.wasm stitching, localStorage persistence, cost tracking, project save/load

Video Export (video-export.ts, 185 lines)

Single scene → direct download. Multiple scenes → FFmpeg.wasm stitching. Re-encodes with libx264 to normalize codecs/resolutions across different models. Mixes audio: voiceover at full volume + background music at 30% volume. Falls back to individual downloads if FFmpeg fails.

Cost Tracker (cost-tracker.ts, 72 lines)

Records every generation cost to localStorage (last 200 entries). Shows total spent + estimates remaining based on pending scenes × model cost/sec. Also has recordCostToDb() for cloud persistence via /api/generations.

Project I/O (project-io.ts, 92 lines)

Export: strips base64 data URLs and blob URLs to keep file size reasonable, saves as .cutagent.json. Import: reads JSON, migrates older versions that may lack new fields (role, trimStart, voiceoverText).

Dependencies

Runtime (4 packages)

  • @fal-ai/client — Direct browser-to-fal.ai generation
  • @ffmpeg/ffmpeg + @ffmpeg/util — Client-side video stitching
  • lucide-react — Icons

Framework

  • Next.js 15 + React 19 — App framework
  • Tailwind CSS 4 — Styling
  • TypeScript 5.8 — Type safety throughout

No Backend Required

  • No database (localStorage for persistence)
  • No auth server (user provides own fal.ai key)
  • No file storage (videos hosted on fal.ai CDN)
  • No queue/worker (fal.ai handles job queue)

Key Design Decisions

  • Adapter pattern for multi-model support (easy to add models)
  • All generation client-side (no server costs)
  • Prompt engine is model-aware (formats differently per model)
  • Style Harness is opt-in (parallel when off, sequential when on)
